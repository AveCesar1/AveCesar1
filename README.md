## Work & Documentation Style

I commit in **verified, self‑contained modules**.  
Most commits represent a fully functional addition and often exceed **600 lines of code**.  
Lighter commits exist only when testing across different environments or when rapid cloud sync is required — never for trivial changes.

Commit messages follow a consistent format:
```
Project X.Y.Z: Description of the change
```

Where **X** is the main module, **Y** the specific area, and **Z** minor adjustments or fixes.  
Example:  
`Artícora 4.2.4: file sharing via chat (AES‑256 E2E)`

This keeps the history clean, searchable, and meaningful.
